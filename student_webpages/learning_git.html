Learning Git

This is very fun! :)

Git is a widely used version control system for software development. And it is extremely useful for developers to keep track of the commits and the changes made in a file.

Let's say, today is october 10th 2015 and I made a change in a project call "little-cats" and there are 3 people working on it. With git, all the people of the team can see the change I made, in a specific time and the reason of why did I made the change. If it's september 23rd of 2020 I can still see the changes and the commit.

  1. o the fist thing you do is to initialize a Git repository by using

    git init

    --A repository is a place where the history of your work is stored --

  2. After you have a repository you have to check the status of the Git by using

    git status

  3. Then you create a file using touch, (I will create the file call hey.html)

    touch hey.html

  4. Then you add something to your file, let's say that you just add the title. To see the change you made on your text editor, you just check your git status again

    git status

    The text will appear red, this means that the file is "untracked" and this is usually because is a newly created file, looking something like this:

    modified: hey.html

  5. So Git is telling you that you just modify the file hey.html, but you don't know what it was changed. To see the changes (or the difference) you made on file, you use

    git diff

  6. Now you can see in red all the changes made, since I change the title, it will look like this

    Learning Git

  7. Now, to tell Git to start tracking changes made on the file, we first need to add it to the staging area using

    git add (and the name of the file)

    git add hey.html

    --Staging area is a place where we can group files together before we "commit" them to Git--

  8. Then we check the status of git

    git status

    and now the text is not longer red, is green and this means that is already commited

    modified: hey.html

  9. Now we add a message or comment for you or other people to see what was changed (maybe you want to see that change in 5 years and the comment will help you to understand what you did there)

    git commit -m "I change the title"

    -m means message

  10. And then if you want to know all the changes that has been made in the file so far you enter

    git log
